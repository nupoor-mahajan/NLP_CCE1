Feature Engineering Pipeline

Prepared by: Hansa Gusaiwal

Clean tokens are still just words - machine learning models need numbers. This pipeline shows how cleaned, tokenized text is converted into numeric features using counting and weighting techniques.

Analogy Once your vegetables are washed and chopped (preprocessed), the next step is turning them into an actual measurable dish - in NLP terms, that means turning words into numbers a model can actually learn from.

Workflow Diagram

graph TD
    A[Clean Tokenized Text] --> B[Build Vocabulary]
    B --> C[Bag of Words Counts]
    B --> D[N-Gram Extraction]
    C --> E[Term Frequency - TF]
    E --> F[Inverse Document Frequency - IDF]
    F --> G[TF-IDF Feature Vectors]
    D --> G

Step-by-Step Breakdown

  1. Clean Tokenized Text: Take the output of the preprocessing pipeline as input.
  2. Build Vocabulary: Collect every unique word across the dataset to form the vocabulary.
  3. Bag of Words Counts: Count how many times each vocabulary word appears in each document.
  4. N-Gram Extraction: Optionally extract sequences of 2 or 3 consecutive words to capture some context.
  5. Term Frequency (TF): Normalize word counts within each document by document length.
  6. Inverse Document Frequency (IDF): Calculate how rare each word is across the whole corpus.
  7. TF-IDF Feature Vectors: Combine TF and IDF to produce the final numeric feature vectors used for modelling.

Why This Pipeline Matters

The quality of features directly decides how well a machine learning model performs. Good feature engineering - like the TF-IDF vectors used in the Amazon reviews sentiment analysis project - often matters just as much as the choice of model itself.
